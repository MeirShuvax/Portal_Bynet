# 📁 מדריך שמירת תמונות בפרודקשן

## איך זה עובד?

### בפיתוח (Development):
- התמונות נשמרות בתיקיית `employee-portal-server/uploads/`
- נגישות דרך: `http://localhost:5000/uploads/filename.jpg`

### בפרודקשן (Production):
- התמונות נשמרות בתיקיית `uploads/` בשרת
- נגישות דרך: `https://your-domain.com/uploads/filename.jpg`
- התיקייה נוצרת אוטומטית אם היא לא קיימת

## הגדרות לפרודקשן

### אפשרות 1: נתיב ברירת מחדל (מומלץ)
הקוד כבר מוכן! התמונות יישמרו אוטומטית בתיקיית `uploads/` בשרת.

**אין צורך לעשות כלום** - זה יעבוד אוטומטית! ✅

### אפשרות 2: נתיב מותאם אישית
אם תרצה לשמור את התמונות בנתיב אחר, הוסף למשתני הסביבה:

```bash
UPLOADS_PATH=/path/to/your/uploads
```

## מה הקוד עושה?

1. **יצירת תיקייה אוטומטית**: הקוד יוצר את תיקיית `uploads/` אם היא לא קיימת
2. **שמירת תמונות**: כל תמונה חדשה נשמרת עם שם ייחודי
3. **הגשת קבצים**: התמונות נגישות דרך `/uploads/filename.jpg`

## דוגמאות:

### תמונות פרופיל:
- נשמרות כ: `profile-1234567890-987654321-username.jpg`
- נגישות דרך: `/uploads/profile-1234567890-987654321-username.jpg`

### תמונות מערכת (PhotoOfWeek):
- נשמרות כ: `1234567890-987654321-originalname.jpg`
- נגישות דרך: `/uploads/1234567890-987654321-originalname.jpg`

## הערות חשובות:

1. **גיבוי**: מומלץ לגבות את תיקיית `uploads/` באופן קבוע
2. **הרשאות**: וודא שלשרת יש הרשאות כתיבה לתיקייה
3. **גודל**: התמונות מוגבלות ל-5MB (ניתן לשנות ב-Multer)
4. **פורמטים**: רק תמונות (image/*) מתקבלות

## בדיקה:

לאחר העלאה, בדוק שהתמונה נגישה:
```
https://your-domain.com/uploads/filename.jpg
```

אם התמונה לא נטענת, בדוק:
- שהתיקייה קיימת
- שיש הרשאות קריאה
- שהנתיב נכון
